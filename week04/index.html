Chapter 8, 12 and 15

Programming applications containing web forms has always been one of the fundamental tasks of JavaScript. One of the main reasons the JavaScript programming language was invented was the need to validate form data directly in the user's browser. In this way, it was avoided to reload the page when the user made mistakes when filling in the forms. Forms are a very common method of interacting with a web page.

One of the safest ways to access the elements of a form is to access the forms on a page through their name (name attribute) or their id attribute. The document object allows direct access to any form through its name attribute.

Forms and their elements can also be accessed using direct node access DOM functions. The following example uses the common document.getElementById () function to directly access a form and one of its elements.

Regardless of the method used to obtain the reference to a form element, each element has the following useful properties for application development:
1. type: indicates the type of element in question. For elements of type input (text, button, checkbox, etc.) it matches the value of its type attribute.
2. form: is a direct reference to the form to which the element belongs. Thus, to access an element's form, you can use a document.getElementById ("element_id"). Form
3. name: gets the value of the XHTML name attribute. Only its value can be read, so it cannot be modified.
4. value:allows reading and modifying the value of the XHTML value attribute. For text fields (input type="text"> and textarea) it gets the text that the user has entered.

The most used events in the handling of forms are the following:
1. onclick: event that occurs when an element is clicked with the mouse. It is normally used with any of the button types that XHTML allows to define (input type="button">, input type="submit">, input type="image").
2. change: an event that occurs when the user changes the value of a text element (input type="text" or textarea). It also occurs when the user selects an option from a drop-down list (select).
3. onfocus: an event that occurs when the user selects an element of the form.
4. onblur: complementary event of onfocus, since it occurs when the user has deselected an element due to having selected another element of the form. Technically, the previous element is said to have "lost focus".

Object-oriented programming.

This form of programming consists of separating the code into objects that have properties and functions, this allows them to be easily modified and reused.

There are 3 main concepts in object-oriented programming (OOP) which are: encapsulation, polymorphism, and inheritance.

Encapsulation is about keeping all programming logic inside an object and making methods available to implement the functionality, without the outside world needs to know how to do it.

Polymorphism means that multiple objects can share the same method, but they also can override shared methods with a more specific implementation.

Inheritance means that we can take an object that already exists and inherit all its properties and methods. Then we can improve its functionality by adding new properties and methods.

The use of constructor functions allows the creation of examples of objects. The constructor function defines properties and methods and by using the keyword this it is saying that when one of these object instances is created, the name property of the object will be equal to the value of the name passed to the constructor call, and the method it is identified by using the parentheses and will also use the value of the name passed to the constructor call.

A constructor function name generally begins with a capital letter - this convention is used to make constructor functions easier to recognize in code.
ES6 introduced class declarations that use the keyword class. These can be used in place of constructor functions.

Within a constructor function or class declaration, the keyword this refers to the object returned by the function.

JavaScript modules

A module is just a file. A script is a module. As simple as that.

Modules can load each other and use special export and import directives to exchange functionality, call functions of one module from another:
• export variable keyword tags and functions that should be accessible from outside the current module.
• import allows the import of functionalities from other modules.

Module characteristics.
Modules always work in strict mode. For example, assigning an undeclared variable will give an error. Each module has its top-level scope. In other words, the top-level variables and functions of a module are not seen in other scripts
